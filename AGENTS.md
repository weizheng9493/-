# AGENTS.md

本文件定义 Codex 和其他智能体在本仓库中的工作规则。所有智能体修改本仓库时必须遵守。

## 语言和风格

- 默认使用中文。
- 保持客观、可验证，不夸大能力或结果。
- 输出面向团队协作，优先清晰、可执行、可追踪。
- 对不确定内容必须标注“待确认”或“推测”，不得写成事实。

## 仓库定位

本仓库不是普通网盘，而是航旅 AI PM 工作流的产品化仓库。所有内容应服务于以下目标：

- 沉淀可复用工作流。
- 维护 Agent Skill、提示词、文档和案例。
- 管理团队成员或智能体生成的产物。
- 通过 PR、审核和版本发布保证质量。

## 修改原则

- 不要直接修改 `main` 分支。
- 每次变更应尽量小而完整，方便 reviewer 审核。
- 不要删除或重写已有内容，除非任务明确要求。
- 修改现有 Agent Skill 时，需要同步检查其引用文件、示例和说明是否仍一致。
- 新增文档应放在合适目录，不要把所有内容堆到根目录。
- 大段 AI 生成内容必须附带来源、用途、风险和人工校验状态。

## 目录使用规则

| 目录 | 使用规则 |
| --- | --- |
| `docs/` | 存放正式产品文档、治理规则、发布说明和团队说明 |
| `workflows/` | 存放可执行工作流，例如 Issue 到 Codex 到 PR 的流程 |
| `prompts/` | 存放可复用提示词，不存放一次性聊天记录 |
| `artifacts/` | 存放团队成员或智能体生成的产物，每个产物必须有 README |
| `examples/` | 存放示例材料 |
| `hanglv-ai-pm-agent-skills/` | 存放 Agent Skill，不随意改名或移动 |

## 智能体产物提交要求

如果提交的是智能体生成的 Markdown、代码、脚本、原型或分析结果，必须包含：

- 提交人。
- 生成工具或智能体。
- 输入来源。
- 产物说明。
- 适用场景。
- 是否已人工校验。
- 风险、假设和待确认点。
- 是否建议进入正式工作流。

推荐放置路径：

```text
artifacts/<提交人或团队>/<YYYY-MM-DD-主题>/
├── README.md
├── output.md
└── source-prompt.md
```

## PR 输出要求

创建或协助创建 PR 时，应说明：

- 本次更新点。
- 为什么需要更新。
- 影响范围。
- 如何验证。
- 风险和待确认点。

## 验证要求

- 文档类修改至少检查链接、目录和 Markdown 格式。
- 脚本或代码类修改必须说明运行命令和结果。
- 工作流规则类修改必须说明对团队使用方式的影响。

## 禁止事项

- 不要把飞书聊天记录、个人隐私、账号密钥、未脱敏客户信息提交到仓库。
- 不要提交 `.env`、token、cookie、私钥或本地配置文件。
- 不要把未经确认的模型输出写成正式规范。
- 不要为了省事绕过 PR 审核流程。
