# Badcase 库

用于沉淀人工批改后的反例。

## 记录模板

```md
## Badcase：<标题>

### AI 原输出

### 问题

### 为什么错

### 正确处理

### 应更新的规则
```

## 当前已知反例类型

### 背景空洞

只写“为提升用户体验和转化”，没有用户原声、数据、竞品或现状问题。

### 无收益测算

只写“预计提升转化”，没有指标、口径、基线或需要补充的数据。

### 用户场景不准确

把退改、报销、价格、客服等不同问题混成一个泛泛的“售后体验差”。

### 边界状态少

只写默认流程，不写价格变化、退款失败、出票中、航司拒开、状态不同步等边界。

### 承诺不存在的能力

无依据承诺锁价、自动赔付、无条件退款、平台承担全部航司责任。
