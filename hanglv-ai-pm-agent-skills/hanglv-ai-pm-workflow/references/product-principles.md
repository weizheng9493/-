# 产品设计原则

来源：用户提供的《产品设计原则》《做需求》以及补充口径。

## 需求判断

1. 深入理解用户：先成为用户，理解用户心理，不等于照搬用户说法。
2. 从用户、数据、竞品、业务反馈、个人体验中发现问题。
3. 判断需求是否符合产品定位、目标、用户场景。
4. 判断需求强度、频率、收益、成本、紧急重要程度。
5. 能不做就不做，只做必须做的。
6. 决定不做什么，往往比决定做什么更重要。
7. 不为少数边缘场景伤害大多数用户主场景。

## 体验原则

1. 用户进来就应该知道可以干什么，以及如何完成任务。
2. 在满足用户需求前提下，尽量自然、简洁。
3. 降低用户理解成本和操作成本。
4. 可以采用渐进式呈现和渐进式填写。
5. 产品内部保持统一，不让用户为同一件事反复学习。
6. 体验不要过快、过激、来回变化。
7. 指标必须体现真实价值，不能只追求表面渗透率。

## 转化原则

1. 聚焦关键路径，尤其是搜索、选择、确认、支付、出票、售后恢复。
2. 尽量降低下单阻塞。
3. 刺激下单欲望，但不能牺牲透明度和信任。
4. 价值优化先于入口强化；价值和流量要匹配。
5. 对交易型产品，可信、透明、可解释比花哨更重要。

## 原型原则

1. 画原型前先想清楚需求背景、目标、收益、用户场景和流程。
2. 不要一上来画原型；复杂需求先画流程图/泳道图。
3. 原型不必过度高保真，但交互必须清晰。
4. 热区明确：按钮、卡片、文字入口分别能否点击要说清。
5. 操作有反馈：提交中、提交成功、失败、已收藏、已选择等。
6. 使用真实、符合场景的文案和数据。
7. 状态完整：默认、加载、空、错误、成功、禁用、异常。
8. 画完后做可用性检查：用户能不能看懂、流程是否顺畅。
