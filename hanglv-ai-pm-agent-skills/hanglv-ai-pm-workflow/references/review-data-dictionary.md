# 点评数据字典

适用于航旅纵横机票点评 CSV。当前样例文件字段：

| 字段 | 含义 | 使用方式 |
| --- | --- | --- |
| `userid` | 用户 ID | 输出时不要展示原值，必要时做匿名化 |
| `objectid` | 评价对象 ID | 当前口径待确认；不能在未确认前断言 861/862 的业务含义 |
| `itemscore` | 1-5 分评分 | 1-2 分视为负向，3 分中性，4-5 分正向 |
| `content` | 用户自由文本评价 | 高价值原声，需脱敏后引用 |
| `createtime` | 评价时间 | 用于时间趋势、版本影响、阶段性问题判断 |
| `merged_labels` | 合并标签 | 可能来自用户选择、模型或人工清洗；口径未知时标注待确认 |

## 当前已观察到的样本特征

来自 `/Users/weizheng/Downloads/机票点评数据明细.csv` 的一次抽样分析：

- 总行数：41020。
- 1-2 分负向样本：6519。
- 带文本的负向样本：2057。
- 高频负向标签包括：退款到账慢、价格太贵、退改结果通知慢、退改价格不清晰、流程复杂难懂、票价不透明、客服响应慢、信息提示不全、报销不方便。

## 使用规则

1. 标签只能作为问题线索，不能直接等同于产品根因。
2. 低分 + 文本原声 + 高频标签，比单独高频标签证据更强。
3. `content` 为空但标签存在时，可纳入量级统计，但不能作为原声证据。
4. 涉及收益或优先级时，优先看低分量级、负向文本密度、是否发生在交易关键路径。
