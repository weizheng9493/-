# 航旅机票业务规则 V1

本文件是 V1 占位规则，后续应由真实业务负责人继续补充。当前输出中凡未确认的规则必须标为待验证。

## 通用边界

1. 机票价格、库存、退改规则必须以航司/供应链实时返回为准。
2. 不要默认承诺锁价、自动赔付、无条件退改或平台兜底差价。
3. 支付、乘机人、退改签、报销等关键节点必须用户确认。
4. 涉及合规、税务、保险、行程单、发票的内容，需要明确责任主体。
5. 航司配合、支付渠道、财务退款、客服响应等问题，不能单纯用页面改版解决。

## 机票链路常见问题

| 问题 | 可能本质 | 方案边界 |
| --- | --- | --- |
| 价格变动/票价不透明 | 实时票价机制解释不足、库存变化、税费/套餐误解 | 可以做解释、更新时间、变价提示、重搜/继续购买；不能无依据承诺锁价 |
| 退改价格不清晰 | 航司规则复杂、展示不完整、结果通知慢 | 可以做费用拆解、规则说明、进度通知；不能承诺航司规则外退款 |
| 报销不方便 | 行程单/发票主体、航司和平台责任、状态不同步 | 可以做路径缩短、状态透明、材料说明；需明确责任主体 |
| 流程复杂难懂 | 页面路径长、信息多、一次性暴露过多 | 可以做渐进式呈现、默认值、关键动作突出 |
| 客服响应慢 | 自助能力不足、人工处理拥堵 | 可以做自助查询、状态通知、升级入口；不能只靠页面承诺解决 |
