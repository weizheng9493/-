# Artifacts 提交说明

`artifacts/` 用来存放团队成员或智能体生成的待沉淀产物。

这些内容可以包括：

- Markdown 文档。
- 分析报告。
- PRD 草稿。
- 竞品分析。
- 代码或脚本。
- HTML 原型。
- Prompt 实验结果。
- 设计验收报告。

## 目录规范

每个提交请使用独立目录：

```text
artifacts/<提交人或团队>/<YYYY-MM-DD-主题>/
├── README.md
├── output.md
└── source-prompt.md
```

如果包含代码或原型，可以增加：

```text
src/
assets/
screenshots/
```

## README 必填内容

```md
# 产物标题

## 基本信息

- 提交人：
- 日期：
- 使用的智能体或工具：
- 输入来源：
- 产物类型：

## 内容说明

请说明这个产物是什么、解决什么问题。

## 适用场景

请说明团队什么时候可以使用它。

## 人工校验状态

- [ ] 未校验
- [ ] 已初步校验
- [ ] 已充分校验

## 风险和待确认

- 

## 后续建议

- [ ] 保留在 artifacts
- [ ] 整理进 docs
- [ ] 整理进 prompts
- [ ] 整理进 workflows
- [ ] 不建议继续使用
```

## 注意事项

- 不要提交密钥、Token、Cookie、未脱敏用户信息或内部敏感数据。
- 不要把未经人工校验的 AI 输出直接当成正式规范。
- 如果产物来自外部资料，需要说明来源。
