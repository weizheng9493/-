# UI 交付说明：舱位权益展示 Demo

## 1. 设计目标

在不增加用户认知负担的前提下，让用户快速理解不同舱位的权益差异。

## 2. 页面结构

```text
航班信息摘要
→ 舱位卡片列表
  → 舱位名称 / 标签 / 价格
  → 核心权益摘要
  → 查看全部权益
  → 选择按钮
→ 底部确认区域
```

## 3. 舱位卡片信息层级

1. 舱位名称和推荐标签。
2. 价格。
3. 核心权益摘要。
4. 展开入口。
5. 选择状态。

## 4. 核心状态

### 默认

- 展示前 3 个 `highlight = true` 或排序靠前的权益。
- 权益以短文本展示，避免大段说明。

### 展开

- 展示全部权益。
- 权益限制信息进入二级说明。
- 可使用轻量分割线区分权益项。

### 选中

- 卡片边框或背景突出。
- 选择按钮变为“已选择”。
- 底部确认区域同步展示已选舱位。

### 不可选

- 卡片整体降低视觉权重。
- 保留权益查看能力，但禁用选择按钮。
- 明确展示不可选原因。

### 权益缺失

- 展示兜底文案：“以航司实际规则为准”。
- 不展示空字段。

## 5. 设计验收清单

- [ ] 舱位名称、价格、标签清晰。
- [ ] 核心权益不超过 3 条，避免信息过载。
- [ ] 展开后能看到完整权益。
- [ ] 选中态足够明确。
- [ ] 不可选态不会误导用户。
- [ ] 权益缺失有兜底文案。
- [ ] 小屏下文本不会溢出。
