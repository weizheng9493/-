# PRD 草稿：舱位选择页权益展示优化

## 1. 背景

舱位选择页是用户从航班方案进入下单前的重要决策页面。不同舱位通常对应不同价格、退改签规则、行李额、选座、餐食、里程等权益。如果权益展示不清晰，用户容易只按价格选择，或在后续订单/售后环节产生预期落差。

本次先用假数据跑通流程，只验证产品、UI、研发和 QA 的协作方式，不接入真实航司权益接口。

## 2. 目标

- 让用户在舱位选择页清楚看到不同舱位的核心权益差异。
- 支持默认、展开、选中、不可选、权益缺失等关键状态。
- 为后续真实接口接入预留字段结构。
- 跑通 Issue 到 PR 的团队协作闭环。

## 3. 非目标

- 不修改真实航司权益规则。
- 不接入真实库存、价格或退改签接口。
- 不影响真实下单链路。
- 不做完整 A/B 实验。

## 4. 用户故事

作为准备购买机票的用户，我希望在选择舱位时快速看清不同舱位的核心权益差异，这样我可以判断是否值得为更高价格支付额外费用。

## 5. 页面范围

当前 Demo 只覆盖舱位选择页。

后续可扩展承接页面：

- 订单填写页。
- 支付确认页。
- 订单详情页。
- 售后退改签页。

## 8. 交互状态

### 8.1 默认状态

- 展示舱位名称、价格、推荐标签。
- 展示 3 个核心权益摘要。
- 如果权益超过 3 个，展示“查看全部权益”。

### 8.2 展开状态

- 展示全部权益。
- 每个权益展示名称、摘要和必要限制。
- 重点权益优先排序。

### 8.3 选中状态

- 舱位卡片高亮。
- 展示“已选择”状态。
- 底部确认区域展示当前舱位核心权益摘要。

### 8.4 不可选状态

- 舱位卡片置灰。
- 展示不可选原因，例如“余票不足”。
- 不允许点击确认。

### 8.5 权益缺失状态

- 如果某权益字段为空，展示“以航司实际规则为准”。
- 不用空白或错误值占位。

## 9. 埋点建议

| 事件 | 触发时机 |
| --- | --- |
| `cabin_benefit_view` | 舱位权益曝光 |
| `cabin_benefit_expand` | 用户展开权益 |
| `cabin_select` | 用户选择舱位 |
| `cabin_confirm` | 用户确认舱位 |
| `cabin_unavailable_view` | 不可选舱位曝光 |

## 10. 验收标准

- 假数据中至少包含 3 个舱位。
- 至少覆盖一个推荐舱位、一个高权益舱位、一个不可选舱位。
- 每个可选舱位至少展示 3 个权益。
- 页面规则能够覆盖默认、展开、选中、不可选、权益缺失状态。
- 假数据可通过校验脚本。

## 11. 风险和待确认

- 真实权益字段来源待确认。
- 不同航司权益口径不一致，需要兜底策略。
- 后续如果承接到订单页，需要确认订单页字段和展示责任。
