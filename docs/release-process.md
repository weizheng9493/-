# 发布流程

## 发布定义

合并到 `main` 代表内容进入主仓库，但不等于正式发布。正式发布需要创建版本号和 Release。

## 版本号建议

使用语义化版本：

```text
v0.1.0
v0.2.0
v1.0.0
```

建议含义：

| 版本变化 | 含义 |
| --- | --- |
| `v0.x.0` | 内部迭代阶段 |
| `v1.0.0` | 团队正式使用版 |
| `v1.x.0` | 新增重要能力或工作流 |
| `v1.x.1` | 修复文档、示例或小问题 |

## 发布前检查

发布前需要确认：

- 所有相关 PR 已合并。
- `CHANGELOG.md` 已更新。
- 重要文档链接有效。
- 目录结构没有明显混乱。
- 新增智能体产物没有敏感信息。
- 如有脚本或代码，验证结果已经记录。

## 发布步骤

1. 更新 `CHANGELOG.md`。
2. 创建发布准备 PR。
3. 审核并合并。
4. 在 GitHub 创建 Release。
5. 填写版本号、标题和更新说明。
6. 在团队渠道同步发布信息。
